Spotify Song Popularity Analysis

Analyzed Spotify song data to understand which audio features are most related to song popularity.

Python pandas matplotlib Jupyter Notebooks
More details

Problem

With so many different audio features available for songs, it can be difficult to understand what actually makes a song popular. The goal of this project was to identify which features have the strongest relationship with popularity.

Approach

I analyzed a dataset of Spotify songs using Python and pandas. I explored relationships between features such as energy, danceability, loudness, tempo, and valence using summary statistics and visualizations to identify patterns.

Results & Impact

The analysis showed that energy and danceability had strong positive relationships with popularity, while tempo and loudness had little effect. This helps highlight which characteristics are more important when predicting song success.

NYC 311 Data Analysis

Analyzed NYC 311 complaint data to identify trends across boroughs and agencies. Focused on understanding complaint volume and resolution times using structured data analysis.

Python pandas SQL Data Cleaning
More details

Problem

NYC 311 generates a large volume of service request data that is difficult to interpret without proper organization. The goal was to analyze patterns in complaints across boroughs and agencies to better understand service demand and efficiency.

Approach

Cleaned and structured raw complaint data, then analyzed it using Python and SQL. Computed metrics such as complaint counts and average resolution times by borough and agency to uncover patterns in service requests.

Results & Impact

Found clear differences in complaint volume and response times across agencies and boroughs. The analysis highlights how data can be used to identify inefficiencies and improve service delivery in large cities.

AWS NYC 311 Data Pipeline

Built a cloud-based data pipeline using AWS S3 and Athena to analyze NYC 311 service request data. Designed a scalable workflow for storing, querying, and extracting insights from large real-world datasets.

AWS S3 AWS Athena SQL Python pandas
More details

Problem

Large-scale public datasets like NYC 311 are difficult to analyze due to their size and unstructured format. The goal was to create an efficient cloud-based system to store, query, and analyze complaint data across agencies and boroughs.

Approach

Stored raw data in AWS S3 and organized it into structured formats for querying with AWS Athena. Used SQL queries to analyze complaint volume, response times, and patterns across agencies. Designed the workflow to be scalable and easy to extend for future analysis.

Results & Impact

Identified trends in complaint distribution and resolution times across NYC agencies. Demonstrated how cloud-based tools can efficiently handle large datasets and support real-time data analysis workflows.
